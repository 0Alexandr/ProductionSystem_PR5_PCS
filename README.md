## Быстрый запуск

### Требования
- [.NET 8.0 SDK](https://dotnet.microsoft.com/download/dotnet/8.0)

### Шаги

```bash
# 1. Перейти в папку проекта
cd ProductionSystem

# 2. Применить миграции (создаст production.db автоматически при запуске)
dotnet run
```

При первом запуске `Program.cs` автоматически вызывает `db.Database.Migrate()` —
база данных `production.db` создаётся рядом с исполняемым файлом
и заполняется тестовыми данными (seed).

Приложение будет доступно по адресу: **http://localhost:5000**

## Реализованный функционал

### Модели базы данных (EF Core + SQLite)
- `Product` — продукты с категорией, временем производства и мин. запасом
- `ProductionLine` — линии со статусом и коэффициентом эффективности
- `Material` — материалы с единицами измерения и мин. запасом
- `ProductMaterial` — связь продукт↔материал (сколько нужно на 1 ед.)
- `WorkOrder` — заказы со статусом, прогрессом и расчётными датами

### Панель управления (/)
- Статистика: всего заказов, в производстве, ожидают, низкий запас, активные линии, продукты
- Таблица последних 5 заказов с прогресс-барами

### Материалы (/Materials)
- Список с цветовой индикацией низкого запаса (красный/зелёный)
- Фильтр «только с низким запасом»
- CRUD (добавить / редактировать / удалить)
- Кнопка «Пополнить» с авторасчётом дефицита

### Продукты (/Products)
- Список с фильтром по категории и поиском по названию
- CRUD с динамическим редактором материалов (добавить / удалить строку через JS)
- Привязка необходимых материалов с количеством на единицу

### Заказы (/Orders)
- Фильтрация по статусу (Ожидает / В работе / Завершён / Отменён) и дате
- Создание с **авторасчётом** срока: `(Кол-во × Время/шт) ÷ Коэф.эфф.`
- **Проверка материалов** при создании (предупреждение о дефиците)
- Кнопки: «Запустить», «Завершить», «Отменить»
- При запуске — списание материалов со склада
- Страница деталей с обновлением прогресса через API
- Просроченные заказы подсвечиваются красным

### Производственные линии (/Lines)
- Карточки линий с визуальным индикатором статуса (пульсирующая точка)
- Кнопки «Запустить» / «Остановить»
- Слайдер коэффициента эффективности (0.5–2.0) с сохранением
- Расписание назначенных заказов на каждой линии
- Прогресс текущего заказа

### REST API
| Метод | URL | Описание |
|-------|-----|---------|
| GET | `/api/materials?low_stock=true` | Список материалов |
| POST | `/api/materials` | Создать материал |
| PUT | `/api/materials/{id}/stock` | Обновить количество |
| GET | `/api/products?category=X` | Список продуктов |
| GET | `/api/products/{id}/materials` | Материалы продукта |
| POST | `/api/products` | Создать продукт |
| GET | `/api/lines?available=true` | Список линий |
| PUT | `/api/lines/{id}/status` | Изменить статус линии |
| GET | `/api/lines/{id}/schedule` | Расписание линии |
| GET | `/api/orders?status=active&date=today` | Фильтрация заказов |
| POST | `/api/orders` | Создать заказ |
| PUT | `/api/orders/{id}/progress` | Обновить прогресс |
| GET | `/api/orders/{id}/details` | Детали заказа |
| POST | `/api/calculate/production` | Расчёт времени производства |

## Расчётная формула

```
ВремяПроизводства = (Количество × ВремяНаЕдиницу) / КоэффициентЭффективности
```

Пример: 10 ед. × 45 мин/шт ÷ 1.2 = **375 минут (6 ч 15 мин)**


## Удаление и создание БД с нуля

1. Удалить вручную .db и папку migrations

2. Создайте одну чистую миграцию

```bash
dotnet ef migrations add InitialFullCreate
```
3. Создайте файл базы данных

```bash
dotnet ef database update
```
